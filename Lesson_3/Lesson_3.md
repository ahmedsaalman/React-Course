## The Problem First

Look at our current `TaskCard` component from the previous lesson:

```jsx
function TaskCard() {
  return (
    <div className="task-card">
      <h4>Sample Task</h4>
      <p>Subject: Mathematics</p>
      <span>Due: Tomorrow</span>
    </div>
  )
}

```

**The Issue:** Every card looks identical. The data is hardcoded, which makes the component useless in a real, dynamic application.

What we actually want is the ability to write this:

```jsx
<TaskCard title="Math Homework" subject="Mathematics" due="Tomorrow" />
<TaskCard title="Read Chapter 4" subject="Physics" due="Monday" />
<TaskCard title="Essay Draft" subject="History" due="Wednesday" />

```

Three different cards generated by one single component. We achieve this using **props**.

## What Are Props?

Props (short for properties) are the mechanism a parent component uses to pass data down to a child component.

**The Mental Model:**
Think of a component like a stamp machine. The machine (the component) always produces the same overall shape. However, you can feed it different colored ink (props) each time. Different input results in a different output, but you are using the exact same machine.

To bring it back to your function analogy:

```javascript
// Regular JavaScript function with parameters
function greet(name) {
  return `Hello ${name}`;
}

greet("Ali");    // Output: Hello Ali
greet("Sara");   // Output: Hello Sara

```

Props are simply parameters for React components.

```jsx
// React Component with props
function TaskCard(props) {
  return <h4>{props.title}</h4>;
}

<TaskCard title="Math Homework" />   // Renders: Math Homework
<TaskCard title="Read Chapter 4" />  // Renders: Read Chapter 4

```

Same component. Different data. Different output.

---

## How Props Work Internally

When you write a component with attributes like this:

```jsx
<TaskCard title="Math Homework" subject="Mathematics" />

```

React automatically bundles all of those attributes into a single JavaScript object and passes it to your function as the first argument.

```javascript
// What React actually passes to the TaskCard function under the hood:
{
  title: "Math Homework",
  subject: "Mathematics"
}

```

That is why you access them inside the component as `props.title` and `props.subject`.

---

## Destructuring Props: Cleaner Syntax

Instead of writing `props.` everywhere in your component, most React developers use JavaScript object destructuring immediately in the function parameters.

**Without destructuring (Verbose):**

```jsx
function TaskCard(props) {
  return (
    <div>
      <h4>{props.title}</h4>
      <p>{props.subject}</p>
    </div>
  )
}

```

**With destructuring (Clean):**

```jsx
function TaskCard({ title, subject, due }) {
  return (
    <div className="task-card">
      <h4>{title}</h4>
      <p>Subject: {subject}</p>
      <span>Due: {due}</span>
    </div>
  )
}

```

> **Note:** Both methods do the exact same thing, but destructuring is cleaner and easier to read. You will see this pattern in almost all professional React codebases.

## Data Flow is One-Directional

This is a critical rule to understand early in React:

**Props only flow downward. Parent to child. Never the other way.**

App  ->  TaskBoard  ->  TaskCard
(passes)      (receives)

**The Mental Model:**
Think of a waterfall. Water only flows down. A lower rock cannot push water back up to a higher rock. Similarly, a child component cannot modify its own props.

> **Crucial Rule:** Props are strictly read-only.

```jsx
function TaskCard({ title }) {
  // INCORRECT: Never mutate props. 
  title = "changed"; 
  
  return <h4>{title}</h4>;
}

```

If data needs to go back up the tree, we handle that using state and event handlers, which we will cover in Lesson 4.

## Putting It Together: Updated Components

Let us upgrade our planner with everything we have learned.

**1. The Child Component (`TaskCard.jsx`)**

```jsx
function TaskCard({ title, subject, due, completed }) {
  return (
    <div className="task-card">
      <h4>{title}</h4>
      <p>Subject: {subject}</p>
      <span>Due: {due}</span>
      <span>{completed ? "Status: Complete" : "Status: Pending"}</span>
    </div>
  )
}

export default TaskCard;

```

**2. The Parent Component (`TaskBoard.jsx`)**

```jsx
import TaskCard from "./TaskCard";

function TaskBoard() {
  return (
    <div className="task-board">
      <h2>Your Tasks</h2>
      
      <TaskCard
        title="Math Homework"
        subject="Mathematics"
        due="Tomorrow"
        completed={false}
      />
      
      <TaskCard
        title="Read Chapter 4"
        subject="Physics"
        due="Monday"
        completed={true}
      />
      
      <TaskCard
        title="Essay Draft"
        subject="History"
        due="Wednesday"
        completed={false}
      />
      
    </div>
  )
}

export default TaskBoard;

```

Now each card renders dynamically based on the data passed to it. We have one reusable component producing entirely different UI blocks.

## Common Beginner Pitfall: Strings vs. Other Data Types

When passing props, pay close attention to the syntax used for strings versus other data types (booleans, numbers, arrays, objects).

```jsx
// INCORRECT: Passing a boolean as a string
<TaskCard completed="false" />
// React receives the literal STRING "false". In JavaScript, a non-empty string is evaluated as truthy. 
// This will incorrectly render "Status: Complete".

// CORRECT: Use curly braces for non-string values
<TaskCard completed={false} />

```

> **Rule of Thumb:** Strings can use standard quotation marks `""`. Everything else (numbers, booleans, arrays, objects, variables) must go inside curly braces `{}`.
